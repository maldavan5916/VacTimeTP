Тема: Проектирование, кодирование, испытание разрабатываемого модуля.

Тестирование программного обеспечения — это процесс проверки соответствия функционирования программы заявленным требованиям. Оно позволяет выявить дефекты, проверить корректность логики, оценить стабильность и удобство использования системы. Цель тестирования — повысить надёжность ПО и минимизировать риски в процессе эксплуатации.

В рамках текущего этапа был разработан тестовый сценарий для проверки модулей учёта поступлений, генерации и печати отчётов. Тестирование проводилось вручную с фиксацией поведения системы и сравнения с ожидаемыми результатами.

Сценарий включал следующие действия:
- добавление новой записи о поступлении с множественным выбором материалов;
- фильтрация данных по дате и складу в окне отчёта;
- проверка правильности расчёта суммы и итогов;
- формирование отчёта и отправка на печать;
- поворот ориентации страницы перед печатью;
- проверка корректности вывода фамилии ответственного лица.

В процессе тестирования были обнаружены следующие отклонения:
1. Ошибочный расчёт итоговой суммы в отчёте:
   - При группировке по складу сумма округлялась неверно (использовалось округление до целых вместо 2 знаков после запятой).
2. Повторное открытие окна отчёта с теми же фильтрами:
   - Не обновлялась информация при повторной попытке сформировать документ — отображался кэш предыдущего состояния.
3. Некорректное отображение фамилии кладовщика:
   - В случае отсутствия привязки сотрудника к складу, внизу отчёта отображалось «null».
4. Неочевидное уведомление об ошибке печати:
   - При отсутствии установленного принтера сообщение не отображалось, система «молча» завершала выполнение команды.

На основании выявленных ошибок был составлен тестовый сценарий для повторной проверки, в который включено:
- контроль округления итогов при разных режимах группировки;
- проверка корректного обновления отчёта после смены параметров;
- обработка пустых значений при отображении ФИО кладовщика;
- тестирование механизма отображения ошибок при сбое печати;
- проверка реакций системы на случайные пользовательские действия (пустые поля, отмена диалогов и пр.).
