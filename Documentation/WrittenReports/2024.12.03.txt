Тема: Выполнение и сопровождение организационных (производственных) задач, разработанных для организации

1. Разработка и доработка моделей данных
Модели данных в проекте определяют структуру информации, используемой для учета материалов, изделий и их характеристик. Основная работа проводилась с классом `Product_Material`:
	1. Расширение функционала класса:
	   - Добавлен вычисляемый атрибут `GetSum`, который рассчитывает стоимость материалов на основе их количества и цены за единицу. 
	     - Это оптимизирует расчет данных непосредственно в модели, исключая необходимость дополнительных операций при формировании отчетов.
	2. Улучшение аннотаций и структуры модели:
	   - Использован атрибут `NotMapped` для свойств, не сохраняемых в базу данных, но необходимых для вычислений или отображения.
	3. Связи между сущностями:
	   - Реализованы связи с сущностями материалов, их единиц измерения и изделий, что обеспечило корректную загрузку связанных данных.

2. Сопровождение и реализация отчётной системы
Функционал отчетности был существенно улучшен для формирования производственных отчетов. Основные изменения включают:
	1. Базовый класс отчета:
	   - Создан базовый класс `BaseReportViewModel`, который предоставляет унифицированный подход для обработки данных, формирования отчетов и их отображения.
	   - Внедрены команды (`ICommand`), такие как:
	     - `RefreshCommand` — обновление данных в отчете.
	     - `PrintCommand` — печать документа.
	     - `RotateDocCommand` — изменение ориентации страницы.
	2. Функции формирования отчета:
	   - Группировка данных:
	     - Данные группируются по изделиям. Для каждой группы отображаются общие и итоговые суммы.
	   - Общий итог:
	     - Добавлен расчет общей стоимости всех материалов, включенных в отчет.
	   - Настройка внешнего вида документа:
	     - Установлены параметры для отображения страницы, такие как размеры (A4), отступы, шрифты и другие стилистические элементы.
	3. Форматирование таблицы:
	   - Реализовано добавление заголовков и строк таблицы с четкой структурой колонок:
	     - Код, наименование изделия, материал, количество, единица измерения, стоимость за единицу, итоговая стоимость.
	4. Интерактивность отчета:
	   - Поддержка динамического обновления отчета при изменении настроек, таких как группировка или отображение итогов.
   - Возможность изменения ориентации документа для удобства печати.

3. Обновление пользовательского интерфейса
Работа с интерфейсом отчетов (XAML) включала:
	1. Добавление новых команд и кнопок:
	   - Кнопки для обновления, печати и изменения ориентации страницы.
	   - Использование иконок из библиотеки MaterialDesignInXaml для улучшения визуального восприятия.

	2. Привязка данных:
	   - Реализована привязка элементов интерфейса к свойствам ViewModel, что обеспечивает отображение сообщений, управление документами и выполнение действий.

	3. Улучшение стиля:
	   - Применены стили Material Design для обеспечения современного и удобного пользовательского опыта.

4. Функции и методы для поддержки производственных задач

Методы модели `Product_Material`
- `GetSum`:
  - Вычисляет общую стоимость материалов.
  - Формула: `Quantity * Material.Price`.

Методы отчетного класса `BaseReportViewModel`
- `Refresh(object? obj)`:
  - Обновляет содержимое отчета, переформировывая документ на основе текущих данных.
- `PrintReport(object obj)`:
  - Вызывает диалог печати и отправляет отчет на печать.
- `SwapPageDimensions(object obj)`:
  - Меняет местами ширину и высоту документа для изменения ориентации.

Методы класса `MaterialUsageReportViewModel`
- `CreateReport()`:
  - Генерирует документ с отчетом, включая заголовок, таблицу с данными, группировку и итоговые значения.
- `CreateGroupedRows`:
  - Формирует строки таблицы для группированных данных.
- `CreateUngroupedRows`:
  - Формирует строки таблицы для не сгруппированных данных.

Заключение
Проведенные работы позволили оптимизировать процессы формирования отчетов и упростить анализ данных. Новые функции обеспечивают:
- Повышение удобства работы пользователей.
- Сокращение времени на формирование отчетов.
- Улучшение представления производственных данных.

Разработанные и сопровождаемые решения успешно интегрированы в общую систему учета организации.
