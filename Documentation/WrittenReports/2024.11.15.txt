Тема: Выполнение и сопровождение организационных (производственных) задач, разработанных для организации.

Для ООО «ВакТайм» необходимо разработать программное средство «Учет выпуска и реализации продукции».  

Техническое задание (ТЗ) — это документ, в котором устанавливаются цели, требования и функциональные особенности разрабатываемой программы, а также порядок и критерии проверки выполненной работы.  

Задачи технического задания для программы:  
1. Создать базу данных для хранения информации о структурных подразделениях предприятия.  
2. Разработать функционал для учета выпуска изделий (название, заводской номер, принадлежность).  
3. Включить в программу учет наличия комплектующих для производства изделий (по складам и материально ответственным лицам).  
4. Реализовать механизм планирования выпуска изделий на определенные периоды (анализ выполнения договорных обязательств).  

Функции технического задания для программы:  
1. Определить требования к структуре базы данных и ее функционалу.  
2. Установить алгоритмы расчета выпуска продукции за любой период.  
3. Разработать требования к интерфейсу программы для удобного добавления, изменения и удаления данных.  
4. Указать формат отчетов и механизм их формирования для печати.  

Пункты технического задания:  
1. Общие положения:  
   - Назначение программы: учет выпуска и реализации продукции предприятия.  
   - Целевая аудитория: сотрудники производства, складов и руководящий состав.  
2. Функциональные требования:  
   - Добавление, изменение и удаление данных о структурных подразделениях.  
   - Ведение учета выпускаемой продукции с возможностью поиска по заводскому номеру, наименованию или комплектующим.  
   - Анализ выполнения плана выпуска продукции за определенные периоды (сравнение плана и факта).  
   - Формирование отчетов по цехам и продукции с возможностью их печати.  
3. Требования к базе данных:  
   - Таблицы для структурных подразделений, перечня изделий, выпуска продукции, комплектующих и складов.  
   - Поля для уникальных номеров изделий, принадлежности к цеху, информации о комплектующих.  
4. Требования к интерфейсу:  
   - Простота и удобство ввода данных.  
   - Фильтры для поиска по ключевым параметрам (наименование, заводской номер).  
   - Формы для планирования выпуска продукции.  
5. Требования к отчетам:  
   - Формирование отчетов по выпуску продукции за указанный период.  
   - Печать списков изделий с указанием зав. номера, комплектующих и соответствующего подразделения.  
6. Технические требования:  
   - Поддержка работы на ОС Windows.  
   - Легкость обновления базы данных.  
   - Обеспечение быстродействия при большом объеме данных.
